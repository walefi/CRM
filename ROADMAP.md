# Regras

Cada etapa deve:

- seguir o PROJECT_RULES.md;
- preservar compatibilidade;
- reutilizar componentes existentes;
- possuir testes;
- atualizar documentação;
- atualizar Swagger;
- atualizar Storybook;
- executar Docker;
- executar Build;
- executar Lint;
- executar Testes;
- respeitar Multi-Tenant;
- respeitar RBAC;
- respeitar Clean Architecture;
- respeitar DDD;
- respeitar SOLID.

---

# Critério de conclusão

Uma etapa somente poderá ser considerada concluída quando:

- Todos os testes passarem.
- O projeto compilar sem erros.
- Docker estiver funcionando.
- Backend estiver funcionando.
- Frontend estiver funcionando.
- Swagger estiver atualizado.
- Storybook estiver atualizado.
- README estiver atualizado.
- Não existirem erros de TypeScript.
- Não existirem erros de Lint.
- A documentação da etapa estiver concluída.
